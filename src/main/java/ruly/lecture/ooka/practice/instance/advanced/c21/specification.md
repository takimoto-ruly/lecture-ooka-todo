# スマート駐車場料金計算システム 仕様書

## 1. 概要
駐車場の利用記録と車両タイプに基づき、最終的な利用料金を算出・表示する。

## 2. 登場エンティティ（Lombok @Value で定義）
- **VehicleDTO**: `plateNumber` (String), `type` (String: "COMPACT", "NORMAL", "LARGE")
- **ParkingRecordDTO**: `usageMinutes` (int: 利用時間), `isWeekend` (boolean: 土日祝フラグ)
- **ParkingRequest**: `vehicle` (VehicleDTO), `record` (ParkingRecordDTO)

## 3. 三層構造の設計要件

### ① Logic層 (計算)
- **責務**: 基本単価、時間、倍率を受け取り、最終金額を算出する。
- **仕様**: 
    - 計算式：`基本単価 * (利用時間 / 30.0) * 倍率` 
    - ※30分単位での計算（時間は `double` として扱う）。
    - 端数処理：1円単位で切り上げ（`Math.ceil`）を行う。

### ② Service層 (業務ルール)
- **責務**: 車両タイプと曜日フラグに基づき、ビジネスルールを適用して合計料金を算出する。
- **仕様**:
    - **基本単価**:
        - COMPACT（軽自動車）: 200円
        - NORMAL（普通車）: 300円
        - LARGE（大型車）: 500円
    - **倍率ルール**:
        - 土日祝（isWeekend=true）の場合、全車両タイプで **倍率 1.5** を適用。
        - 平日の場合、倍率は **1.0**。
    - **割引ルール**:
        - 利用時間が 180分（3時間）を超える場合、Service層で最終合計額から **200円を差し引く**（ただし合計がマイナスになる場合は0円とする）。

### ③ Controller層 (制御・バリデーション)
- **責務**: バリデーション、Service呼び出し、メッセージ整形、異常値チェック。
- **仕様**:
    - **バリデーション**: リクエスト、車両、記録のいずれかが `null` なら `"Invalid Data"` を返す。
    - **入力チェック**: `usageMinutes` が 0 以下の場合は `"Invalid Time"` を返す。
    - **高額アラート**: 合計料金が 10,000円 を超える場合は `"Contact Staff: [金額]"` を返す。
    - **正常出力**: それ以外は `"Price: [金額] JPY"` と返す。
